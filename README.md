___막대그래프___


___막대그래프 색 지정___
```R
barplot(ds, main="favorite season", col="blue") #막대 색 지정
```

___막대별로 색을 다르게 지정___

### ___히스토그램___
  * 막대 그래프와 비슷한 그래프로, 그룹이 명시적으로 존재하지 않는 수치형 자료의 분포를 시각화할 때 사용한다.
  * hist 함수를 사용해 작성한다
  * 일반적으로 막대 사이에 간격이 있으면 막대그래프, 간격이 없으면 히스토그램으로 구분한다.
  * 막대그래프는 막대의 면적이 의미가 없지만 히스토그램은 막대 면적의 의미가 있다.
```R
head(cars)
dist <- cars[,2]                    #자동차 제동거리
dist
hist(dist,                          #data
     main="Histogram for 제동거리", #제목
     xlab="제동거리",               #X축 레이블
     ylab="빈도수",                 #Y축 레이블
     border="blue",                 #막대 테두리 색
     col="green",                   #막대 색
     las=2,                         #X축 글씨 방향
     breaks=5)                      #막대 개수 조절
```

___구간별 빈도수를 도수분포표 형태로 출력___

### ___조건문___

  * c <- ifelse(a>b, a, b) 는 a가 b 보다 크면 a를 a가 더 작으면 b를 c에 저장하라는 의미이다.

___if-else의 반복___
  * if문 뒤에 else if를 중첩 사용해 조건별 명령문을 작성할 수 있다.

### ___반복문___
___for문___
  * for문의 반복 범위는 반복 변수에 할당할 값을 모아둔 벡터로 이 벡터의 길이만큼 반복된다.
  * for(반복 변수 in 반복 범위) { 반복할 명령 } 으로 작성한다.

___while문___
  * while문은 비교 조건을 만족하는 동안 {}안의 명령문을 반복 실행한다.
  * while(비교조건) { 반복할 명령 } 으로 작성한다.

___apply 계열 함수___
  * 반복 작업의 대상이 매트릭스나 데이터프레임의 행이나 열일 때 앞의 반복문 대신 apply 함수를 사용할 수 있다.
  * R에선 대규모 데이터 분석 시, 처리 속도 문제 때문에 되도록 for문이나 while문을 사용하지 않는 것을 권장한다.
  * apply 함수를 사용하면 처리 속도를 향상시킬 수 있다.
  * apply(데이터셋, 행/열 방향 지정, 적용 함수) 로 작성한다.
  * apply와 유사한 함수로 lapply, sapply, tapply, mapply 등이 있다.


### ___사용자 정의 함수___
  * R은 사용자 스스로 함수를 만들 수 있도록 지원하는데, 그것이 바로 사용자 정의 함수이다.
  * 함수명 <- function(매개변수 등록) { 실행할 명령 / return(함수 실행 결과) } 로 작성한다.
  * 작성된 함수는 코드 차제를 한 번 실행해 주면 R은 생성한 함수를 기억하고 필요할 때 호출해 사용할 수 있다.

___매개변수에 기본값 설정___
  * 함수 생성 시, function() 안의 매개변수에 =값 을 입력해주면 함수 사용시 매개변수가 없으면 해당 값을 기본값으로 동작한다.

___여러 개의 값을 반환하는 경우___
  * 여러 개의 값을 반환하려면, list 함수를 이용해 여러 개의 결과값을 하나로 묶고 그 list를 반환하면 된다.

___사용자 정의 함수의 저장과 재실행___
  * 자주 사용하게 될 사용자 정의 함수는 파일에 모아두었다가 필요시 호출해서 사용한다.
  * source 함수로 저장된 R 파일 내부의 함수를 실행한다.

___조건에 맞는 데이터 위치 찾기___
  * 원하는 데이터가 어디에 위치하는지 알고싶을 때, which, which.max, which.min 함수를 사용한다.
  * which 함수의 매개변수에 값을 넣으면, 해당 값이 몇번 째 인덱스에 있는지 알려준다.
  * which.max는 최댓값의 인덱스. which.min은 최솟값의 인덱스를 알려준다.
  * 매트릭스, 데이터프레임에서 사용 시, 원하는 값의 인덱스 뿐 아니라 값의 이름도 알 수 있다.

  * 변수명은 값이 무엇인지 알 수 있도록 의미있는 단어를 사용하는 것을 권장한다.  
  * 자주 사용하는 함수의 이름은 변수명으로 사용하는 것을 권장하지 않는다.  

___변수에 저장될 수 있는 값의 종류___
  * 숫자형(정수, 실수, 0 포함)
  * 문자형(작은따옴표, 큰따옴표로 묶어서 표현)
  * 논리형(TRUE, FALSE)
  * 특수 값(Null, NA, NaN, Inf, -Inf)

### ___벡터___
  * 여러 개의 값을 한꺼번에 저장하는 기능
  * 다른 프로그래밍 언어의 배열과 같음 (벡터는 인덱스가 1부터 존재한다.)

___벡터 만들기___
  * 변수 <- c(데이터1, 데이터2, 데이터3)
  * c 함수는 여러개의 값을 하나로 묶어준다.
  * 하나의 벡터에는 동일한 자료형의 값이 저장되어야 한다.

___연속적인 숫자 저장___
  * 변수 <- 시작값:종료값

___일정한 간격의 숫자 저장___
  * 변수 <- seq(시작값, 종료값, 간격)

___반복되는 숫자 저장___
  * 변수 <- rep(반복대상값, times=반복횟수)
  * times 대신 each를 쓰면 대상 값들을 각각 정해진 횟수만큼 반복할 수 있다.

___벡터 값에 이름 붙이기___
  * 벡터를 만든 사람은 벡터의 저장된 데이터들의 의미를 알지만 제 3자는 그 의미를 알기 어렵다.
  * names() 함수를 이용해 각각의 값들의 이름을 붙힐 수 있다.

___벡터의 원소값 확인하기___
  * 벡터는 인덱스가 1부터 존재한다.
  * 벡터이름[인덱스] 로 출력한다.
  * 벡터이름[c(인덱스1, 인덱스2, 인덱스3)] 등으로 여러개의 값을 출력할 수 있다.
  * 벡터이름[-인덱스] 로 해당 인덱스를 제외한 값을 출력할 수 있다.
  * names()함수로 값의 이름을 저장한 경우, 해당 이름을 통해 값을 출력할 수 있다.

___벡터의 원소값 변경하기___
  * 벡터이름[인덱스] <- 변경값 로 다른 값으로 변경할 수 있다.
  * 벡터이름[c(인덱스1, 인덱스2)] <- 변경값1, 변경값2 로 여러개의 다른 값으로 변경할 수 있다.
  * 벡터이름 <- c(변경값) 으로 해당 벡터의 내용을 통째로 다른 값으로 변경할 수 있다.

### ___함수___

___함수의 매개변수___
  * 매개변수 : 함수의 입력값을 받는 변수  
  * 함수의 정의에 맞추어 매개변수를 입력하면 정의된 결과값을 얻을 수 있다.

___매개변수 입력___
  * 함수(매개변수명 = 매개변수값)으로 입력받는다.

___매개변수 생략___
  * 함수(매개변수값) 으로 생략 가능하다.
  * 여러개의 매개변수가 존재할 때 모두 생략하거나 일부만 생략할 수 있다.
  * 매개변수값까지 통째로 생략할 수 있는 선택적 매개변수와 반드시 입력해야하는 필수 매개변수가 있다.

통계엔 R과 파이썬을 주로 사용한다.

1. ___사용자 커뮤니티가 탄탄___

    * 초보자를 위한 자료도 많이 있다.
    * 한글 자료들도 증가하는 추세이다.  
<br>
2. ___다양한 패키지 제공___  
    * 데이터 분석에 사용되는 함수들을 종류별로 묶어 패키지 형태로 제공한다.  
    * 데이터 분석에 필요한 거의 모든 기능을 제공한다.  
    * 최신 이론이 발표되면 바로 R패키지가 만들어져 신속하게 활용 가능하다.  
<br>
3. ___미적, 기능적인 통계 그래프 제공___  
    * 데이터 분석에서 분석 결과를 시각적으로 표현하는 건 매우 중요하다.  
    * R에선 ggplot이라는 패키지를 통해 예쁘고 기능적인 그래프를 쉽게 작성 가능하다.  
<br>
4. ___편리한 프로그래밍 환경___  
    * R 프로그래밍을 위한 통합 개발 환경으로 R스튜디오가 제공되어, 모든 작업을 스튜디오 안에서 처리 가능하다.  
    * 프로그램 작성, 실행, 수정 등 여러 작업을 수행하려면 보다 편리한 작업 환경이 필요하다.

### ___R 스튜디오의 화면 구성___
1. ___소스영역___  
     * 소스창 : R 명령문을 작성 및 실행하는 영역이다.  
    명령문은 컴퓨터에게 일을 시키기 위한 작업 문장인데, R 프로그램은 명령문들의 집합으로 이루어진다.  
    <- : 대입 연산자

2. ___콘솔영역___  
    * 콘솔창 : 명령문의 실행 과정과 결과가 표시되는 영역이다.
    * 터미널창 : 명령 프롬프트와 동일한 기능을 제공한다.

3. ___환경영역___  
    * 환경창 : 각종 변수나 자료구조의 내용을 보여주는 영역이다.
    * 히스토리창 : 실행한 명령문, 결과, 패키지 설치 등을 보여주는 영역이다.
    * 커넥션창 : 데이터 관리를 위한 서버를 연결하는 창이다.
    * 튜토리얼창 : R을 따라하며 배울 수 있는 창이다.

4. ___파일영역___  
    * 파일창 : 윈도우의 파일 탐색기와 동일한 역할을 하는 영역이다.
    * 플롯창 : R 명령문으로 작성한 그래프가 표시되는 영역이다.
    * 패키지창 : 설치된 R 패키지들의 목록을 볼 수 있고, 새 패키지를 다운로드 할 수 있는 영역이다.
    * 도움말창 : 특정 함수에 대한 도움말을 찾아볼 수 있는 창이다.
    * 뷰어창 : 분석의 결과가 숫자가 아닌 이미지 형태일 때 표시되는 창이다.

### ___명령문 실행___
 * 소스창 상단의 실행 버튼을 클릭하면 커서가 위치하고 있는 라인의 명령문이 실행된다.  
 * ctrl + enter를 누르면 커서가 위치하고 있는 라인의 명령문이 실행된다.  
 * 명령문을 블록 설정하고 실행하면 블록 부분의 명령문이 모두 실행된다.  
 * 불완전한 명령문 입력 시 콘솔창에 + 표시가 나타나고 나머지 명령문이 입력될 때까지 기다린다.  
 * 잘못된 명령문 입력 시 붉은색의 에러 메세지가 나타난다.  
 * ; 기호로 한 줄에 여러개의 명령문을 작성할 수 있다.

### ___산술연산자___
* R은 컴파일을 통해 .exe 형태의 실행 파일을 만들지 않고 명령문을 실행하는 인터프리터 방식이다.
* +, -, *, /, %%, ^ 등의 연산자가 있다.

### ___주석___
* 주석은 #으로 작성한다.

### ___산술연산 함수___
* log, sqrt, max, min, abs 등의 함수가 있다. 

### ___패키지___

* 패키지를 R에서 사용할 수 있도록 하는 작업은 로딩이라고 한다.
