___1차원 자료와 2차원 자료___
  * 1차원 자료 : 단일 주제에 대한 값들을 모아 놓은 것을 말한다. (벡터, 배열)
  * 2차원 자료 : 복수 주제에 대한 값을 모아 놓은 자료를 말한다. (테이블, 매트릭스)

___범주형 자료와 수치형 자료___
  * 범주형 자료 : '분류'의 의미를 갖는 값들로 구성된 자료를 말한다. (문자로 표현, 산술연산 X)
  * 수치형 자료 : 값들이 크기를 가지며 산술연산이 가능한 자료를 말한다.

### ___벡터 연산___
___벡터에 대한 산술연산___
  * 벡터에 대한 산술연산은 벡터 안에 포함된 값들 하나하나에 대한 연산으로 바뀌어 실행된다.

___벡터와 벡터의 연산___
  * 벡터와 벡터 간의 산술연산은 벡터간 대응되는 위치에 있는 값들끼리의 연산으로 바뀌어 실행된다. (벡터의 길이와 데이터 타입이 같아야한다.)

___벡터에 적용 가능한 함수들___
  * sum, mean(값들의 평균), median, max, min, var(분산), sd(표준편차), sort, range, length 등

___벡터에 논리연산자 적용___
  * <, <=, >, >=, ==, !=, |, &

### ___팩터와 리스트___
  * 팩터와 리스트는 벡터와 같이 R에서 1차원 형태의 자료를 저장하는 자료구조이다.

___팩터___
  * 문자형 데이터가 저장되는 벡터의 일종으로, 저장되는 문자 값들이 어떠한 종류를 나타내는 값(범주형 자료)일 때 사용한다.
  * 문자형 벡터를 만둔 후 factor 함수를 이용해 변환하여 만든다.
  * 팩터는 저장된 문자값을 숫자로 바꾸어 분석 작업에 활용할 수 있다.

___리스트___
  * 자료형이 다른 값들을 한곳에 저장하고 다룰 수 있는 수단이다.
  * 벡터 생성 후 list 함수를 이용해 리스트를 생성한다.

### ___매트릭스와 데이터프레임___
  * 매트릭스와 데이터프레임은 2차원 자료를 저장하기 위한 대표적인 자료구조이다.
  * 매트릭스에 저장되는 모든 자료의 데이터 타입이 동일하지만, 데이터프레임에는 서로 다른 타입의 데이터가 저장될 수 있다.
  * 여러 개의 벡터를 모은 것이 매트릭스와 데이터프레임이다.
  * 2차원 데이터는 테이블 형태로 표현된다. (가로줄 : 행, 관측값 / 세로줄 : 열, 변수)

___매트릭스 생성___
  * 매트릭스는 보통 숫자로만 구성된 2차원 자료를 저장하고 처리하는데 사용된다.
  * matrix 함수를 사용해 매트릭스를 생성한다.
  * 해당 함수의 각 매개변수는 매트릭스에 저장될 값, 매트릭스의 행의 수(nrow), 매트릭스의 열의 수(ncol)를 의미한다.
  * 매트릭스는 기본적으로 열 방향으로 데이터가 채워진다. (행 방향으로 채우려면 매개변수에 byrow=T 를 추가한다.)
  * cbind, rbind 함수로 여러 백터들을 묶어 새로운 매트릭스를 만들 수 있다. (길이가 같아야 한다.)

___매트릭스에서 값 추출___
  * 행과 열을 지정하는 2개의 인덱스 값을 사용해 값을 추출한다.
  * 메트릭스에 있는 여러 개의 값을 동시에 지정할 수 있다.

### ___변수___  
  * 변수 : 프로그램 내에서 특정 값을 저장하는 공간 (변경 가능)  
  * 변수 선언 : 변수명 <- 값  

___변수 내용 확인 방법___  
  1. 변수명 입력  
  2. print(변수명)  
  3. cat("변수 : ", 변수명)  

___변수의 특징___  
  1. 변수의 값은 바뀔 수 있다.  
  2. 하나의 변수에는 하나의 값만 저장 가능하다.  
  3. 변수와 변수의 연산은 변수에 저장된 값들의 연산으로 실행된다.  

___변수명의 작명 규칙___
  * 첫 글자는 영문자나 마침표로 시작한다. (일반적으론 영문자로 시작.)  
  * 두 번째 글자부턴 영문자, 숫자, 마침표, 밑줄을 사용할 수 있다.  
  * 변수명에서 대문자와 소문자는 별개의 문자로 취급한다.  
  * 변수명 중간에 빈 칸을 넣을 수 없다.  
  * 변수명은 값이 무엇인지 알 수 있도록 의미있는 단어를 사용하는 것을 권장한다.  
  * 자주 사용하는 함수의 이름은 변수명으로 사용하는 것을 권장하지 않는다.  

___변수에 저장될 수 있는 값의 종류___
  * 숫자형(정수, 실수, 0 포함)
  * 문자형(작은따옴표, 큰따옴표로 묶어서 표현)
  * 논리형(TRUE, FALSE)
  * 특수 값(Null, NA, NaN, Inf, -Inf)

### ___벡터___
  * 여러 개의 값을 한꺼번에 저장하는 기능
  * 다른 프로그래밍 언어의 배열과 같음 (벡터는 인덱스가 1부터 존재한다.)

___벡터 만들기___
  * 변수 <- c(데이터1, 데이터2, 데이터3)
  * c 함수는 여러개의 값을 하나로 묶어준다.
  * 하나의 벡터에는 동일한 자료형의 값이 저장되어야 한다.

___연속적인 숫자 저장___
  * 변수 <- 시작값:종료값

___일정한 간격의 숫자 저장___
  * 변수 <- seq(시작값, 종료값, 간격)

___반복되는 숫자 저장___
  * 변수 <- rep(반복대상값, times=반복횟수)
  * times 대신 each를 쓰면 대상 값들을 각각 정해진 횟수만큼 반복할 수 있다.

___벡터 값에 이름 붙이기___
  * 벡터를 만든 사람은 벡터의 저장된 데이터들의 의미를 알지만 제 3자는 그 의미를 알기 어렵다.
  * names() 함수를 이용해 각각의 값들의 이름을 붙힐 수 있다.

___벡터의 원소값 확인하기___
  * 벡터는 인덱스가 1부터 존재한다.
  * 벡터이름[인덱스] 로 출력한다.
  * 벡터이름[c(인덱스1, 인덱스2, 인덱스3)] 등으로 여러개의 값을 출력할 수 있다.
  * 벡터이름[-인덱스] 로 해당 인덱스를 제외한 값을 출력할 수 있다.
  * names()함수로 값의 이름을 저장한 경우, 해당 이름을 통해 값을 출력할 수 있다.

___벡터의 원소값 변경하기___
  * 벡터이름[인덱스] <- 변경값 로 다른 값으로 변경할 수 있다.
  * 벡터이름[c(인덱스1, 인덱스2)] <- 변경값1, 변경값2 로 여러개의 다른 값으로 변경할 수 있다.
  * 벡터이름 <- c(변경값) 으로 해당 벡터의 내용을 통째로 다른 값으로 변경할 수 있다.

1. ___사용자 커뮤니티가 탄탄___

2. ___다양한 패키지 제공___  
    * 데이터 분석에 사용되는 함수들을 종류별로 묶어 패키지 형태로 제공한다.

<br>
4. ___편리한 프로그래밍 환경___  
    * R 프로그래밍을 위한 통합 개발 환경으로 R스튜디오가 제공되어, 모든 작업을 스튜디오 안에서 처리 가능하다.  
    * 프로그램 작성, 실행, 수정 등 여러 작업을 수행하려면 보다 편리한 작업 환경이 필요하다.

### ___R 스튜디오의 화면 구성___
1. ___소스영역___  
     * 소스창 : R 명령문을 작성 및 실행하는 영역이다.  
    명령문은 컴퓨터에게 일을 시키기 위한 작업 문장인데, R 프로그램은 명령문들의 집합으로 이루어진다.  
    <- : 대입 연산자

2. ___콘솔영역___  
    * 콘솔창 : 명령문의 실행 과정과 결과가 표시되는 영역이다.
    * 터미널창 : 명령 프롬프트와 동일한 기능을 제공한다.

3. ___환경영역___  
    * 환경창 : 각종 변수나 자료구조의 내용을 보여주는 영역이다.
    * 히스토리창 : 실행한 명령문, 결과, 패키지 설치 등을 보여주는 영역이다.
    * 커넥션창 : 데이터 관리를 위한 서버를 연결하는 창이다.
    * 튜토리얼창 : R을 따라하며 배울 수 있는 창이다.

4. ___파일영역___  
    * 파일창 : 윈도우의 파일 탐색기와 동일한 역할을 하는 영역이다.
    * 플롯창 : R 명령문으로 작성한 그래프가 표시되는 영역이다.
    * 패키지창 : 설치된 R 패키지들의 목록을 볼 수 있고, 새 패키지를 다운로드 할 수 있는 영역이다.
    * 도움말창 : 특정 함수에 대한 도움말을 찾아볼 수 있는 창이다.
    * 뷰어창 : 분석의 결과가 숫자가 아닌 이미지 형태일 때 표시되는 창이다.

### ___명령문 실행___
 * 소스창 상단의 실행 버튼을 클릭하면 커서가 위치하고 있는 라인의 명령문이 실행된다.  
 * ctrl + enter를 누르면 커서가 위치하고 있는 라인의 명령문이 실행된다.  
 * 명령문을 블록 설정하고 실행하면 블록 부분의 명령문이 모두 실행된다.  
 * 불완전한 명령문 입력 시 콘솔창에 + 표시가 나타나고 나머지 명령문이 입력될 때까지 기다린다.  
 * 잘못된 명령문 입력 시 붉은색의 에러 메세지가 나타난다.  
 * ; 기호로 한 줄에 여러개의 명령문을 작성할 수 있다.

### ___산술연산자___
* R은 컴파일을 통해 .exe 형태의 실행 파일을 만들지 않고 명령문을 실행하는 인터프리터 방식이다.
* +, -, *, /, %%, ^ 등의 연산자가 있다.

### ___주석___
* 주석은 #으로 작성한다.

### ___산술연산 함수___
* log, sqrt, max, min, abs 등의 함수가 있다. 

### ___패키지___
* R의 패키지는 기능별로 함수를 나누어 놓은 라이브러리 개념  
* 패키지를 R에서 사용할 수 있도록 하는 작업은 로딩이라고 한다.
